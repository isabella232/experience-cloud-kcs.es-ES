<b>Entorno</b>

AEM 6.x

<b>Causa</b>

La sesión JSP se agrega demasiado tarde en la pila de inclusión de script de sling.  Dado que se han escrito suficientes bytes en la respuesta, el motor del servlet Jetty limpia automáticamente el búfer de respuesta antes de añadir la cookie JSESSIONID. Esto provoca un error.


## Resolución


Hay algunas soluciones posibles para este problema:

1. Si no necesita sesiones JSP en la aplicación, asegúrese de que este código se añada en la parte superior de cada archivo jsp:

`%@ page session="false"%`
2. Modifique el código JSP de la aplicación para añadir la sesión JSP en un archivo jsp que se haya cargado anteriormente en el proceso de resolución de sling.  Para ello, establezca el indicador de sesión JSP en true:

`%@ page session="true"%`
3. O aumente el tamaño del búfer de respuesta del motor del servlet Jetty para evitar el vaciado automático prematuro. El tamaño del búfer de respuesta se puede establecer en la configuración OSGi de Jetty (http://host:port/system/console/configMgr/org.apache.felix.http). Puede aumentar esto de forma segura hasta 200000.
